# Sprout

Sprout 是一个跑在 Windows 上的专注白名单工具。你开始一次倒计时会话后，系统只允许你预先放行的窗口继续停留在前台；如果切到别的窗口，守卫进程会立刻把它最小化并记一次违规。浏览器场景优先读取 ActivityWatch 的本地数据，按当前标签页域名判断是否允许。

## 当前实现

桌面端采用 Electron，守卫逻辑跑在独立 Node 进程里。主窗口关闭后，托盘和守卫仍继续运行。ActivityWatch 不是强依赖，离线时软件会自动退化成纯窗口级专注模式。当前界面已经拆成规则、专注、历史、设置四个主 tab，并内置 Windows 系统安全白名单，避免把资源管理器、任务栏等关键系统窗口误判成违规。

## 启动

先在 `/E:/Projects/Sprout` 执行 `npm install`，然后执行 `npm start`。如果你本机已经装了 ActivityWatch 并且浏览器扩展在上报数据，浏览器会自动启用域名级限制；否则浏览器会退化成整窗级。

## 关键交互

规则页里先把要保留的窗口切到前台，再点击加入当前活跃窗口；域名规则支持输入完整 URL 或裸域名，程序会自动规范化成域名。分类规则支持用正则或 `|` 分隔关键词做匹配，只要标题、进程名、路径、域名或 URL 命中，就直接放行。设置页可以修改 Markdown 历史记录目录，默认写到用户文档目录下的 `Sprout/history`。历史页按天展示 Markdown 文件，每次专注会话都会生成一个 `##` 小节。会话开始后，允许列表会锁定；结束会话必须长按退出按钮，避免误触。
